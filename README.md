# 🤖 AI Browser Agent (MVP)

Автономный браузерный агент нового поколения. Использует мощь современных LLM для понимания веб-интерфейсов, планирования действий и автономного выполнения задач в браузере Firefox через Playwright.

---

## 🚀 Основные возможности

- **Умное планирование**: Разложение сложных задач пользователя на последовательность атомарных действий.
- **Анализ DOM**: Собственный движок извлечения интерактивных элементов (`DOMExtractor`) с последующим сжатием состояния страницы для экономии токенов.
- **Мульти-провайдерность**: Поддержка Anthropic Claude, OpenAI, **Routeway** (через OpenAI-совместимый API) и Mock-клиента для тестов.
- **Безопасность прежде всего**: Встроенная проверка на деструктивные действия (удаление, оплата, отправка форм) с запросом подтверждения у пользователя.
- **Сохранение сессий**: Поддержка `cookies.txt` (JSON/Netscape) для работы в авторизованных зонах без повторного входа.
- **Визуальный контроль**: Работа в head-режиме (по умолчанию) для наблюдения за действиями агента в реальном времени.

---

## 🛠 Архитектура системы

- **`src/agent/orchestrator.py`**: "Мозг" системы. Управляет жизненным циклом задачи и взаимодействием компонентов.
- **`src/agent/planner.py`**: Логика принятия решений. Формирует контекст для LLM и интерпретирует ответы.
- **`src/browser/`**: Контроллер браузера на базе Playwright и набор высокоуровневых действий (`click`, `type`, `navigate`).
- **`src/dom/`**: Модули `extractor.py` (поиск кнопок/ссылок) и `summarizer.py` (создание текстового резюме страницы).
- **`src/llm/client.py`**: Универсальный клиент для работы с различными AI провайдерами с защитой от Unicode-ошибок под Windows.
- **`config/settings.py`**: Централизованное управление конфигурацией через `.env`.

## 📦 Установка

### 1. Требования
- Python **3.12+**
- Windows, macOS или Linux (полная поддержка WSL2)

### 2. Клонирование и окружение
```bash
# Клонируйте репозиторий
git clone <url-репозитория>
cd agent

# Создайте и активируйте виртуальное окружение
python -m venv venv
# Для Windows:
venv\Scripts\activate
# Для Linux/macOS:
source venv/bin/activate

# Установите зависимости
pip install -r requirements.txt
```

### 3. Настройка Playwright
Для работы агента необходимо скачать бинарные файлы браузера:
```bash
playwright install firefox
```

## ⚙️ Настройка (.env)

Скопируйте пример конфига и заполните свои ключи:
```bash
cp .env.example .env
```

| Переменная | Описание |
|------------|----------|
| `LLM_PROVIDER` | `mock` / `claude` / `openai` / `routeway` |
| `ROUTEWAY_API_KEY` | Ключ для Routeway.ai (рекомендуется для тестов) |
| `ROUTEWAY_MODEL` | Модель для Routeway (напр. `gpt-4o-mini` или `gpt-oss-120b:free`) |
| `ANTHROPIC_API_KEY` | Для прямого использования Claude |
| `OPENAI_API_KEY` | Для моделей GPT-4/GPT-3.5 |
| `BROWSER_HEADLESS` | `false` (видеть браузер) или `true` (фоновый режим) |
| `BROWSER_SLOW_MO` | Задержка в мс между действиями (для реалистичности) |

## 🎬 Запуск

Запустите основного агента командой:
```bash
python run.py
```

Агент предложит ввести задачу в интерактивном режиме.

**Примеры задач:**
- *"Зайди на youtube.com и найди видео с котиками"*
- *"Открой google.com и узнай текущий курс доллара"*

## 🛡 Безопасность и Unicode
Проект адаптирован для работы под **Windows**. Логгер автоматически настраивает UTF-8 кодировку для консоли и файлов, предотвращая ошибки `UnicodeEncodeError` при выводе эмодзи и спецсимволов.

Агент:

1. Запустит браузер.
2. Извлечёт DOM\-элементы, создаст compressed state.
3. Попросит LLM выбрать действие (navigate / click / type\_text / ...).
4. Выполнит действие, обновит историю.
5. Повторит цикл, пока не будет вызван `task_complete`.

Все шаги и прогресс печатаются в консоль, основные события логируются в `logs/orchestrator.log` и другие файлы логов по именам логгеров.

## Безопасность

Модуль `src/utils/security.py`:

- Отслеживает потенциально деструктивные действия:
  - имена инструментов (delete, remove, pay, checkout, send, post и т\.д.);
  - содержимое селекторов;
  - текст, который вводится.
- При обнаружении деструктивного действия запрашивает явное подтверждение в консоли:
  - `Продолжить? (y/n)`

Без подтверждения действие не выполняется.


## Ограничения MVP

- Планировщик разбивает задачу в одну подзадачу (вся задача = одна `SubTask`).
- Поддерживаются только базовые инструменты:
  - navigate, click, type\_text, extract\_text, wait, screenshot, task\_complete, ask\_user.
- Нет сложной логики многошагового планирования и сохранения сессий между запусками задач, кроме cookies.
- Взаимодействие с пользователем происходит только через консоль (input/print).
